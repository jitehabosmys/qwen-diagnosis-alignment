# 两张显卡的使用策略

## 1. 目标

当前有两类可选机器：

- `RTX 5060 Ti 16GB`
- `RTX 4090 24GB`

我们的目标不是“哪张卡都做同一件事”，而是让两张卡承担不同阶段的职责：

- 便宜卡先验证流程、跑第一轮真实训练
- 更强的卡再承担正式主实验

这样做可以降低试错成本，同时保留最后结果的质量。

## 2. 总体结论

### 2.1 推荐分工

推荐的分工是：

- `5060 Ti 16GB`
  - 用来做流程验证
  - 用来跑第一轮真实 `SFT`
  - 用来调通 LLaMA-Factory 配置
  - 用来验证数据格式、训练日志、checkpoint 保存、推理样例

- `4090 24GB`
  - 用来跑正式主实验
  - 用来尝试更大的基座模型
  - 用来跑更完整的 `SFT`
  - 后续如果继续做 `DPO`、`vLLM`、对比实验，也优先放在 4090 上

### 2.2 为什么不直接上 4090

虽然 4090 更强，但如果一开始就直接上 4090：

- 一旦数据格式错了，会浪费更贵的 GPU 时间
- 一旦模板、参数、路径没配好，会把调试成本放到最贵的阶段
- 第一轮实验通常是“发现问题”而不是“产出最终结果”

所以更合理的方式是：

**先在便宜但还够用的卡上把链路跑通，再把贵卡留给正式实验。**

## 3. `RTX 5060 Ti 16GB` 策略

### 3.1 适合做什么

这张卡明显强于 `P4 8G`，已经不只是 smoke test 机器。

它适合：

- 验证 LLaMA-Factory CLI 训练链路
- 验证 `dataset_info.json` 和数据集格式
- 跑一版真实的 LoRA / QLoRA `SFT`
- 验证 `430` 条最终数据的训练可行性
- 跑小模型观察 loss 是否下降
- 保存并加载 adapter，做基础推理检查

### 3.2 不建议做什么

这张卡不建议承担：

- 最终的大模型主实验
- 比较激进的上下文长度
- 更高风险的显存边缘实验
- `vLLM` 正式部署验证
- 复杂多轮对比实验

### 3.3 推荐基座模型

优先顺序建议如下：

1. `Qwen2.5-0.5B-Instruct`
2. `Qwen2.5-1.5B-Instruct`

推荐理由：

- 中文能力更合适
- 模型较小，显存压力可控
- 更适合第一轮 SFT 验证
- 便于快速看训练是否有收敛迹象

### 3.4 推荐训练阶段

建议把 5060 Ti 上的训练拆成两步：

#### 第一阶段：流程验证

- 数据量：先抽 `20~50` 条
- 目的：
  - 验证数据格式
  - 验证 tokenizer / template
  - 验证保存 checkpoint
  - 验证 loss 正常

#### 第二阶段：小规模真实训练

- 数据量：可以直接用 `430` 条全量
- 模型：`Qwen2.5-0.5B` 或 `1.5B`
- 目的：
  - 跑通第一轮有意义的 SFT
  - 观察模型是否学会结构化输出
  - 验证训练耗时和基本效果

### 3.5 训练目标

在 5060 Ti 上的核心目标不是“训出最终最强模型”，而是：

- 跑通完整链路
- 发现配置问题
- 发现数据问题
- 判断当前任务和格式是否真的能学

换句话说：

**5060 Ti 的任务是把风险暴露出来。**

## 4. `RTX 4090 24GB` 策略

### 4.1 适合做什么

4090 是当前真正的主力卡。

它适合：

- 正式 `SFT`
- 更高质量的最终 checkpoint 训练
- 更大模型实验
- 更长上下文 / 更稳妥 batch 配置
- 后续 `DPO`
- `vLLM` 推理验证

### 4.2 推荐基座模型

如果 5060 Ti 上流程已经验证通过，4090 上可以更积极一些。

建议优先尝试：

- `Qwen2.5-1.5B-Instruct`
- 如果流程很顺，再考虑更大的 Qwen 基座

是否直接上 4B，要看你实际的：

- 显存占用
- LLaMA-Factory 配置
- 是否使用 QLoRA
- 上下文长度

第一次正式实验仍建议先保守，不要一上来把所有风险叠满。

### 4.3 推荐任务

4090 上的实验目标应当是：

- 产出真正要保留的 SFT 结果
- 对比不同基座或不同参数
- 评估全量 `430` 条训练效果
- 为后续 `DPO` 打基础

### 4.4 适合承担的后续工作

如果第一轮 SFT 没问题，4090 还可以继续承担：

- `DPO`
- 更长 step 的 SFT
- 对比不同学习率 / batch / cutoff_len
- `vLLM` 加载与推理验证

## 5. 两张卡的推荐执行顺序

推荐顺序如下：

1. 用 `5060 Ti 16GB` 跑一版最小训练
2. 确认数据格式、训练日志、输出结果都正常
3. 在 `5060 Ti` 上跑第一轮完整小模型 SFT
4. 复盘：
   - 是否 loss 正常下降
   - 输出 JSON 是否变稳定
   - 是否需要调数据或模板
5. 再切到 `4090 24GB`
6. 跑正式主实验

## 6. 为什么不再考虑 `P4 8G`

`P4 8G` 仍然能做非常轻量的 smoke test，但在当前条件下：

- 5060 Ti 只贵很少
- 显存从 `8GB` 提升到 `16GB`
- 架构更现代
- 更有机会直接跑真实训练

所以如果能选 5060 Ti，就没必要再把时间花在 P4 上。

## 7. 当前建议

如果现在就准备开始：

- 优先租 `RTX 5060 Ti 16GB`
- 先用它做第一轮真实 SFT
- 不要一开始直接上 UI，优先用 CLI
- 流程跑顺后，再切到 `4090 24GB` 做正式主实验

## 8. 一句话版本

可以把这两张卡理解成：

- `5060 Ti 16GB`：调试卡 / 首轮验证卡 / 小规模真实训练卡
- `4090 24GB`：正式训练卡 / 后续 DPO 卡 / 推理验证卡
