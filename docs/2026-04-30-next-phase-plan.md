# 2026-04-30 下一阶段实验路线

这份文档记录 `2026-04-30` 时点对当前项目路线的判断与下一阶段执行建议。

## 1. 当前阶段结论

截至目前，`0.5B` 路线已经完成了它最重要的任务：

- 跑通了 `SFT` 训练链路
- 证明了当前数据可以学到结构化 JSON 输出
- 建好了自动规则评测流程
- 建好了 `LLM-as-a-judge` 流程
- 建好了高置信 `DPO` 候选对采集链路

因此，`0.5B` 的价值已经比较清楚：

- 它是一个**流程验证模型**
- 它适合做低成本快速实验
- 它适合验证数据、训练、评测、judge、DPO 采集这一整条链路

但它未必是后续偏好优化最值得投入的主力模型。

## 2. 为什么现在不急着在 `0.5B` 上做正式 DPO

当前不建议立刻在 `0.5B` 上投入正式 `DPO`，核心原因是：

- `0.5B` 的基础能力本身偏弱
- 它当前学到的提升，很大一部分可能还是：
  - 更像训练集输出格式
  - 更像目标 schema
  - 更像当前数据集风格

这并不等价于：

- 学到更好的诊断偏好
- 学到更稳的证据约束
- 学到更强的工程判断

换句话说，如果现在直接基于 `0.5B` 做大规模 `DPO`，一个明显风险是：

**模型可能主要在放大“格式和风格对齐”，而不是提升真正有价值的诊断偏好。**

所以当前更合理的判断是：

- `0.5B` 先停在“流程验证模型”的定位
- 不把它当作正式偏好优化主力模型

## 3. 下一阶段主线

接下来更合理的主线是：

1. 先做 `3B/4B` 的 `SFT`
2. 再做推理评测 + `LLM judge`
3. 再决定是否进入正式 `DPO`

这条顺序比“直接在 `0.5B` 上做 DPO”更稳，因为它先回答了一个更重要的问题：

**更强的模型在完成 SFT 后，是否已经具备足够好的基础内容质量，值得继续做偏好优化。**

## 4. 对 `5090 Ti` 的使用建议

如果当前可用机器是 `5090 Ti`，建议优先承担下面两类任务：

### 4.1 `3B/4B` smoke training

目的：

- 验证显存是否稳定
- 验证训练吞吐
- 验证 checkpoint 保存
- 验证推理能否顺利加载 adapter

这一步的重点不是效果，而是：

- 探清 `3B/4B` 在当前机器上的承载边界

### 4.2 `3B/4B` 完整 SFT 压力测试

如果 smoke 稳定，就继续跑一轮完整 `SFT`，至少回答：

- 训练是否能完整跑完
- loss 是否正常下降
- JSON/schema 是否依旧稳定
- 相比 base，内容质量是否有明显提升

## 5. 为什么 `3B/4B SFT` 比 `0.5B DPO` 更优先

原因很直接：

1. `3B/4B` 的基础语言能力和诊断潜力更强
2. 如果基础模型表达力不够，再做偏好优化收益会被上限限制
3. `DPO` 更适合在“基础回答已经还不错”的前提下做精修

因此当前最合理的资源投入方式是：

- 先把更强模型的 `SFT` 跑出来
- 再看它的输出质量是否值得进入偏好优化阶段

## 6. 现有 judge 与 DPO 采集链路的定位

当前已有的：

- 自动规则评测
- pairwise judge
- `dpo_pairs.jsonl`
- `dpo_pairs_high_confidence.jsonl`

这些都不应该废弃，而应当保留为下一阶段的基础设施。

它们当前的正确定位是：

- 先作为评测工具
- 同时作为偏好数据采集工具
- 暂时不急于立即触发正式 DPO 训练

更具体地说：

- judge 流程继续保留
- 高置信偏好对继续积累
- 等 `3B/4B SFT` 结果出来后，再从更强模型的输出中收集更高质量的偏好对

## 7. 进入 DPO 之前的建议门槛

建议至少满足下面几个条件，再正式进入 `DPO`：

1. `3B/4B` 的 `SFT` 结果已经稳定优于 base
2. 提升不只体现在 schema 上，也体现在内容质量上
3. judge 高置信偏好对已经积累到可用规模
4. 失败样本分析表明，当前主要瓶颈已经从“基础能力不足”转向“偏好质量仍可优化”

如果这些条件不满足，就不建议急着上 DPO。

## 8. 建议执行顺序

当前推荐的实际执行顺序是：

1. 在 `5090 Ti` 上跑 `3B` smoke
2. 跑 `3B` 完整 `SFT`
3. 用现有自动规则评测 + pairwise judge 做验证
4. 判断 `3B` 是否已经值得进入 `DPO`
5. 如果 `3B` 结果很好，再考虑：
   - 是否上 `4B`
   - 是否进入正式 `DPO`

## 9. 当前最稳的路线判断

可以把当前项目分成两个阶段理解：

### 第一阶段：流程验证阶段

代表模型：

- `0.5B`

目标：

- 跑通训练与评测链路
- 验证数据与 schema
- 验证 judge 与 DPO 采集流程

### 第二阶段：主实验阶段

候选模型：

- `3B`
- `4B`

目标：

- 做更有意义的 `SFT`
- 做真实内容质量评测
- 再决定是否进入偏好优化

## 10. 一句话结论

当前最合理的节奏是：

**先把 `3B/4B SFT -> 推理评测 -> LLM judge` 跑出来，再决定是否进入正式 `DPO`；`0.5B` 继续保留为流程验证模型，而不是偏好优化主力模型。**
