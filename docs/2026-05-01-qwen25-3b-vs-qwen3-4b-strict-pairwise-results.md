# 2026-05-01 Qwen2.5-3B vs Qwen3-4B Strict Pairwise Judge 结果

这份文档记录的是 `2026-05-01` 基于 `LLM-as-a-judge` 对两组 strict prompt LoRA 输出做的 pairwise 比较结果。

本轮比较对象：

- `qwen25_3b_strict_json_prompt_lora`
- `qwen3_4b_strict_json_prompt_lora`

这轮比较的目标是：

- 判断在 strict 协议下，`4B` 是否仍然显著优于 `3B`
- 判断继续 scale up 到 `4B` 是否依然有明确收益
- 判断后续主实验路线是否应继续保留 `4B` 作为更强候选

## 1. 为什么这轮比较特别重要

和之前 `0.5B -> 3B` 的比较不同，这轮比较发生在一个更“干净”的条件下：

- 两边都已经完成了 strict prompt 路线的 SFT
- 两边都已经具备很强的协议遵循能力
- 比较重点不再是“谁更会 obey schema”
- 而更接近“谁在明确协议下内容质量更高”

因此，这轮 pairwise judge 对“是否继续 scale up 值得”有更强的解释力。

## 2. 总体结果

本轮 `48` 对样本的 judge 汇总结果如下：

- `qwen25_3b_strict_json_prompt_lora` 胜：`4`
- `qwen3_4b_strict_json_prompt_lora` 胜：`38`
- `tie`: `6`

去掉 `tie` 之后的胜率：

- `qwen25_3b_strict_json_prompt_lora = 9.52%`
- `qwen3_4b_strict_json_prompt_lora = 90.48%`

这个结果非常强，说明 `4B` 的优势不是边缘性的，而是明显的。

## 3. judge 来源分布

- `openai_pairwise_judge = 46`
- `gate_schema_validity = 2`

这说明：

- 绝大多数样本都已经进入真正的内容比较
- `4B` 的优势不是靠大量 schema gate 直接“赢格式”
- 而主要是靠 judge 在内容层面给出的偏好

这点非常关键，因为它意味着：

**这轮比较比“谁更会输出合法 JSON”更接近真实能力比较。**

## 6. 如何解释“winner 差距很大，但分数差距是中等”

这是这轮结果里最有价值的地方之一。

表面上看：

- `4B` 几乎在 winner 上全面压过 `3B`

但分数上看：

- 平均分只提升了 `+0.1304`

这说明：

- `4B` 并不是少数样本特别强，把整体拉高
- 而是在很多样本上都“略优但稳定优”

这是一种非常健康的优势形式：

- 胜率大幅领先
- 但不是靠极端样本刷分
- 说明 `4B` 的提升是普遍性的，而不是偶然性的

## 7. 这轮结果说明了什么

### 7.1 第一层结论

**在 strict 协议下，`4B` 相比 `3B` 依然有明确内容质量优势。**

这说明：

- 把协议讲清楚之后
- `4B` 仍然比 `3B` 更强

所以 `4B` 的提升不是“只会更会猜 schema”，而是内容层面也继续变强。

### 7.2 第二层结论

**至少到 `4B` 这一档，继续 scale up 仍然是值得的。**

因为当前结果并没有显示：

- `4B` 只是勉强赢一点
- 或者平均分几乎不再增长

相反，它仍然表现出：

- winner 上非常强的领先
- 平均分上中等幅度的继续提升

### 7.3 第三层结论

尽管 `4B` 明显强于 `3B`，但这并不意味着：

- 问题已经彻底解决

因为从平均分看：

- `4B` 也只是 `3.2391 / 5`

也就是说：

- `4B` 已经比 `3B` 更像一个合格系统
- 但它还没有到“完全成熟、无须继续优化”的程度

## 8. 对后续路线的影响

这轮结果会把路线进一步收束成：

1. `strict prompt` 作为正式主线是合理的
2. `4B` 目前是更强的主实验候选
3. 但是否继续更大规模，还需要继续看边际收益
4. 同时也说明后续 `DPO` 仍然有意义，因为当前平均分还没有到非常高的水平

## 9. 当前最合理的判断

如果现在要在 `3B` 和 `4B` 之间选一个更值得后续继续投入的模型，我会选：

- `4B strict lora`

原因不是它只在某一个维度上略强，而是：

- 胜率明显更高
- 五个内容维度全面占优
- 平均分也继续抬升

## 10. 一句话结论

这轮 `48` 对 strict pairwise judge 表明：

**`Qwen3-4B-Instruct-2507 + LoRA` 在 strict 协议下以 `38:4:6` 显著优于 `Qwen2.5-3B-Instruct + LoRA`，并在证据贴合、根因质量、可执行性、缺失信息质量和整体工程质量上全面占优。尽管平均分提升属于中等幅度（`+0.1304`），但这足以说明至少到 `4B` 这一档，继续 scale up 仍然具有明确收益。**
